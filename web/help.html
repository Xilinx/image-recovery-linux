Overview

The AMD System Boot Image Recovery Tool (SBIRT) is a web-based interface designed to help you recover and update boot images on AMD embedded systems. This tool provides an intuitive way to manage system firmware and image recovery through either Ethernet or USB connections.

Note: This tool requires appropriate permissions and network access to function properly. Ensure you have the necessary credentials before proceeding with recovery operations.

System Information

The System Information section displays critical details about your system board and current boot image status.

System Board Information

Shows hardware details including:

  • Board name and model
  • Serial number
  • Manufacturing date
  • Hardware revision

Boot Image Current Status

Displays the current state of boot images on the system:

  • Active boot partition (A or B)
  • Boot image version
  • CRC32 checksum validation
  • Image integrity status

Carrier Card Information

If applicable, displays information about any attached carrier cards or expansion modules.

Ethernet Recovery

Use Ethernet Recovery to upload boot images and WIC (Wic Image Creator) images directly from your computer to the system via network connection.

Boot Image Recovery (.bin files)

Step-by-step process:

  1. 1Click the "Boot Image" button
  2. 2Select a .bin file from your computer
  3. 3Verify the file information displayed
  4. 4Click the "Upload" button to begin the transfer
  5. 5Monitor the progress bar for upload status
  6. 6Wait for confirmation of successful upload
Warning: Interrupting the upload process may corrupt the boot image and render the system unbootable. Ensure a stable network connection before starting the upload.

WIC Image Recovery (.wic or .wic.xz files)

WIC images are used for eMMC or SD card recovery. The process is similar to Boot Image recovery:

  1. 1Click the "WIC Image" button
  2. 2Select a .wic or .wic.xz file
  3. 3If using a compressed image (.xz), optionally select a BMAP file for faster flashing
  4. 4Review file information and click "Upload"
  5. 5Monitor progress and wait for completion
BMAP Files: Block map files (.bmap) significantly speed up the flashing process by only writing data blocks that contain actual data, skipping empty blocks. Using a BMAP file is highly recommended for large WIC images.

Upload Progress

During the upload process:

  • A progress bar shows the upload percentage
  • File size and transfer rate may be displayed
  • Do not close the browser or navigate away from the page
  • Do not disconnect the network cable

USB Recovery

USB Recovery allows you to select and apply boot images stored on USB drives connected to the system.

Preparing USB Device

  1. Format a USB drive with a FAT32 or ext4 filesystem
  2. Copy the desired .bin (boot image) or .wic/.wic.xz (WIC image) files to the USB drive
  3. If using compressed WIC images, include the corresponding .bmap file
  4. Safely eject the USB drive from your computer
  5. Connect the USB drive to the target system

USB Recovery Process

  1. 1Navigate to the "USB Recovery" section
  2. 2Click either "Boot Image" or "WIC Image" button
  3. 3A file picker modal will appear showing available USB devices
  4. 4Select the desired file from the list
  5. 5Click "Open" to confirm selection
  6. 6If needed, select the BMAP file for WIC images
  7. 7Click "Upload" to begin the recovery process
Refresh Button: If your USB device is not detected, use the refresh button in the file picker to rescan for connected USB devices.
